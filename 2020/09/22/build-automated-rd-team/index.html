build automated rd team

2020-09-22

現在整個 RD 團隊跑起來了
基本上是全部遠端協作
每天早上十點開約十五分鐘的 Standup
早上期間所有人看 PR
中午吃完飯後開始進行開發
下午三點半自由參與約半小時的 Teatime 幹話
還可以順便討論不懂的地方
沒意外下午六點下班
但其實我也不知道大家什麼時候下班
每週三大家都會到公司
早上就是跟需求方一起半小時的 Sprint Review
接著採隔週 Review 制度
一週是約一小時的 Code Review，討論 Closed PR
一週是約一小時的 Retrospective Review，討論 Scrum
接著星期三下午就來做下次 Sprint 的 Planning 投點
完成一次的 Sprint
目前正在嘗試每月分享會，輪流由兩個人負責
分享主題不限（非技術可），約十五分鐘至半小時，順便吃吃喝喝
或許可以延伸到其他的部門就是了
最後是每季的1on1
此次導入了 1 Page Self-review
請大家在一對一面談前先做一頁式的個人回顧
大幅深入瞭解了每個工程師的狀況與期望，大家也願意講更多
更棒的是能夠讓大家回顧成效，改進不足，訂定目標

現在 RD 團隊正採用 Discord 作為協作工具
發現使用 Discord 後大家討論得更頻繁了
可能是調性的關係，讓大家覺得比較好講幹話
而且還沒有訊息量的限制，權限也可以很細緻
開會的品質很穩定，操作上也很便利，分享螢幕也容易
由於還剛開始使用，還沒有更多心得
之後可以再多多紀錄這部分

會打這篇除了紀念自己好像開始能帶起團隊
讓整個團隊自動的運作起來
沒有囤積的PR，沒有加班，沒有閉門造車
有明確的進度預估，有透明的交流空間
覺得很開心能建立這樣的產品團隊

如今，反思自己的定位
是不是又沒有自己的位置的了呢
一直以來好像都是這樣呢
建立團隊的過程中，不斷的弱化自己的存在
當團隊起來後，自己接近不存在了
「只有我不存在的城市」
常常會想著自己到底在做些什麼呢
明明應該要是件很開心的事情
卻在背後有種莫名的失落
最大失落感應該是在自己吧
明明大家都很開心的工作
自己卻好像沒到這麼樣的開心
明明造就了不錯的環境與團隊
自己卻好像不在裡面
執行者的成果容易被看見
管理者的成果容易被忽略
希望能度過心中的檻
與大家同樂
