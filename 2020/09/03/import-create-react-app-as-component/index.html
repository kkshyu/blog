import create-react-app as component

2020-09-03

繼上次分享兩個 create-react-app（core & example）
example 可以直接把 core 的 當作 component 來使用
實際上把我們的專案拿去使用發現問題重重啊～～
主要是 CRA 的 webpack 做了太多事情了
我們在 build 的時候僅使用 tsc 來編譯，有些語法就會出錯
像是引入 SVG 的部分
CRA 2.0 使用 import { ReactComponent as MyIcon } from './images/icon.svg'
並像這樣子使用 <MyIcon />
因為 CRA 2.0 的 webpack 有裝 svgr 的套件
所以才能夠把 svg 轉成 ReactComponent
原本的寫法應該為 import MyIcon from './images/icon.svg'
並像這樣子使用 <img src={MyIcon} />
MyIcon 會是以字串（圖片編譯後的路徑）方式被引入
於是有幾種方式可以解決：

  1. 想辦法使用 CRA 的 webpack（eject 太難懂、tsconfig 無法 emit 中間 js 檔案）
  2. 自己模擬 CRA 寫 webpack 放入 svgr 等項目（有夠難放）
  3. 把 code 改成不使用其他 webpack 插件（改 SVG 等寫法）
    後來採用的是 3.，因為其他都失敗了ＱＱ
    總共有 200+ 檔案要調整，希望 CRA 未來可以支援客製化的 tsconfig
    這樣就可以透過修改 noEmit: false 來讓他產出編譯出的 js（尚未 bundle 成 chunk）
